# Credo Paper Branch Pilot Plan

## Executive Summary

Before building the Credo platform, we will run a **low-tech pilot** to validate core concepts with real participants. This pilot uses existing tools (Notion, Google Docs, spreadsheet) to test:

1. **Branching workflow** — How do people fork and merge ideas?
2. **Contribution tracking** — How do we measure value?
3. **Credibility system** — Does reputation tracking work?
4. **Egoless collaboration** — Can people contribute without ego?

---

## Pilot Design

### Topic Selection

**Chosen Topic:** *"How can young adults with challenging backgrounds contribute meaningfully to research?"*

**Why this topic:**
- Aligns with Credo's mission (at-risk youth)
- Limited existing research
- Practical application
- Can generate real insights

### Participants

**Target:** 5-7 participants

**Profile:**
- 2-3 people with research experience
- 2-3 people from target demographic (young adults with challenging backgrounds)
- 1 facilitator (you, Kristaps)

**Roles:**
| Role | Responsibility |
|------|---------------|
| Facilitator | Set up tools, moderate discussions, synthesize |
| Researchers | Contribute insights, review others |
| Synthesizer | Identify patterns, create summary |

### Tools Used

| Tool | Purpose | Cost |
|------|---------|------|
| Notion | Branch document, knowledge base | Free |
| Google Sheets | Contribution tracking, credibility | Free |
| Google Meet | Weekly discussions | Free |
| Discord/Signal | Async communication | Free |

## Workflow

### Phase 1: Setup (Days 1-7)

**Day 1-2: Tool Setup**
- [ ] Create Notion workspace
- [ ] Create branch document template
- [ ] Create contribution tracking spreadsheet
- [ ] Set up Discord/Signal group

**Day 3-5: Recruitment**
- [ ] Reach out to potential participants
- [ ] Explain the pilot
- [ ] Get consent (see consent form below)
- [ ] Schedule weekly calls

**Day 6-7: Onboarding**
- [ ] Send welcome message with instructions
- [ ] First orientation call (30 min)
- [ ] Assign initial roles

### Phase 2: Branch Creation (Days 8-14)

**Day 8-9: Initial Contributions**
- [ ] Each participant writes 2-3 paragraphs on the topic
- [ ] Post to Notion branch document
- [ ] Label contributions by type (research, comment, synthesis)

**Day 10-11: Review**
- [ ] Others read and endorse (1-click Google Form)
- [ ] Identify gaps and questions

**Day 12-14: First Synthesis**
- [ ] Facilitator creates first synthesis document
- [ ] Identify "branching points" — different perspectives

### Phase 3: Active Contribution (Days 15-21)

**Day 15-17: Branching**
- [ ] If major disagreement → create sub-branch
- [ ] Each sub-branch has champion
- [ ] Document different perspectives clearly

**Day 18-19: Cross-Pollination**
- [ ] Participants read other branches
- [ ] Endorse good ideas from other branches
- [ ] Propose merges (combine ideas)

**Day 20-21: Second Synthesis**
- [ ] Facilitator updates synthesis
- [ ] Identify consensus points
- [ ] Document remaining disagreements

### Phase 4: Synthesis & Review (Days 22-28)

**Day 22-24: Final Contributions**
- [ ] Last chance for new insights
- [ ] All endorsements finalized
- [ ] Credibility scores calculated

**Day 25-26: Final Report**
- [ ] Create "Branch Report" document
- [ ] Summary of findings
- [ ] Credibility leaderboard (anonymous)
- [ ] Lessons learned

**Day 27-28: Debrief**
- [ ] Final call — discuss what worked/didn't
- [ ] Collect feedback
- [ ] Decide: Continue? Build platform? Pivot?

## Credibility Tracking

### Spreadsheet Structure

```
Columns:
- Participant (anonymous ID)
- Contribution Link
- Type
- Weight
- Endorsements Received
- Total Score
- Tier
```

### Tier Thresholds

| Tier | Score Range |
|------|-------------|
| Newcomer | 0-9 |
| Contributor | 10-29 |
| Trusted | 30-79 |
| Elder | 80+ |

### Weekly Tracking

| Week | Update |
|------|--------|
| 1 | Initial scores |
| 2 | First recalculation |
| 3 | Second recalculation |
| 4 | Final scores + leaderboard |

---

## Endorsement System

### How to Endorse

1. Read someone's contribution
2. Fill out 30-second Google Form:
   - Your anonymous ID
   - Contribution ID you're endorsing
   - Quality rating (1-5)
3. Score = (Quality Rating) × (Contribution Weight)

### Endorsement Limits
- Max 5 endorsements per person per week
- Can't endorse your own contributions
- Can't endorse without reading full contribution

## Risk Mitigation

### Risk: Low Participation
**Mitigation:** Over-recruit (aim for 7), have backup participants

### Risk: One Person Dominates
**Mitigation:** Cap contributions per person per week

### Risk: Conflict
**Mitigation:** Facilitator intervenes, "agree to disagree" option

### Risk: Drops Out
**Mitigation:** Document progress, can continue with fewer

---

## Post-Pilot Decision Framework

### If Pilot Succeeded (≥80% metrics met)
→ Proceed to platform development
→ Use learnings to inform SPEC.md

### If Pilot Partially Succeeded (50-79% metrics met)
→ Run pilot #2 with improvements
→ Identify specific pain points to address

### If Pilot Failed (<50% metrics met)
→ Re-evaluate core assumptions
→ Consider alternative approaches
→ May need different topic or participants
