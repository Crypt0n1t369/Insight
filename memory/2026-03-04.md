#### 2. Platform Architecture Defined

**Core Modules:**
- Identity Service (anonymous auth, wallet optional)
- Branch Service (git-like versioning for research)
- Contribution Service (research, comment, review, synthesis)
- Credibility Service (weighted scoring, tiers)
- Governance Service (quadratic voting, proposals)
- Moderation Service (reports, appeals)

**6-Layer Architecture:**
1. Client Layer (Next.js, PWA)
2. API Gateway (REST, GraphQL, WebSocket)
3. Application Layer (services)
4. Data Layer (PostgreSQL, Redis, IPFS)
5. Identity Layer (anonymous, ZK proofs)
6. Infrastructure (Supabase, Vercel)

#### 3. Database Schema Complete
- 9 core tables: users, branches, contributions, endorsements, proposals, votes, credibility_log, reports, user_warnings
- Functions: credibility calculation, tier mapping, quadratic weights
- Triggers: branch stats, endorsement credibility, auto tier update
- Row Level Security policies

#### 4. MVP Scope Defined (57 hours)
- Weeks 1-2: Foundation (Next.js, Supabase, auth)
- Weeks 3-4: Core features (branches, contributions, credibility)
- Launch: Vercel deployment + Paper Branch pilot
