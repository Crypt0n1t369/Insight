**Mission:** Explore unconventional paths, controversial ideas, and contrarian approaches to Credo's design

## CORE MISSION

Most platforms follow the same patterns. This agent explores the edges, the weird, the uncomfortable questions. What could we do that others aren't?

---

## RESEARCH AREAS (Contrarian)

### A. Anti-Patterns to Consider

1. **What if credibility was hidden?** 
   - Stack Exchange shows numbers
   - Reddit shows karma
   - What if ONLY you see your credibility?
   - Would this reduce gaming? Or remove motivation?

2. **What if there was no branching?**
   - Single canonical version
   - Edit wars = feature, not bug
   - "The discourse IS the product"
   - Reddit model but for structured content

3. **What if forks were permanent?**
   - No merging ever
   - Full divergence allowed
   - Community selects via migration
   - Like language dialects

4. **What if governance was random?**
   - Sortition (random selection)
   - Jury duty model
   - Rotate who decides
   - No credibility needed to vote

5. **What if identity was fully fluid?**
   - Multiple wallets per person
   - "Split personality" reputation
   - Merge/split identities
   - Dark patterns: reputation laundering

### B. Uncomfortable Questions

1. **Should we exclude bad actors by default?**
   - Risk assessment before join
   - Background as data
   - Or: pure innocence until proven guilty

2. **Should credibility decay?**
   - "Use it or lose it"
   - Prevents credential hoarding
   - Or: devalues long-term contribution

3. **Should we pay contributors?**
   - Micro-payment per contribution
   - Sustainability concerns
   - Or: keep it volunteer, preserve motivation

4. **Should we integrate with real-world identity?**
   - LinkedIn/glassdoor style
   - Better for "real" impact
   - Or: purity of anonymity

5. **Should branches have "beliefs"?**
   - Explicit ideology alignment
   - Pre-filter by values
   - Or: pure neutrality

### C. Obscure Research Topics

1. **Dunbar's Number in Online Communities**
   - How big can a "branch" get before it breaks?
   - Research: optimal community sizing

2. **Asymmetric Information Games**
   - Contributor knows more than reviewer
   - How to design for hidden quality

3. **Attention Economics**
   - Scarcity = attention
   - How to distribute fairly?

4. **Ostrom's Commons Principles**
   - Governing the commons digitally
   - Research: Elinor Ostrom's design principles

5. **Memetics & Idea Propagation**
   - Ideas as viruses
   - How to measure "idea fitness"?

6. **Psychographics over Demographics**
   - Don't ask who, ask how they think
   - Value alignment over identity

### D. Platform Comparisons (Not Mainstream)

| Platform | Unconventional Feature | What We Can Learn |
|----------|----------------------|-------------------|
| **4chan** | No accounts, total anonymity, minimal moderation | Raw contribution without credential |
| **Hacker News** | Minimal UI, no gamification, slow | Quality over engagement |
| **Lobste.rs** | Invite-only, tag-focused | Quality curation |
| **LessWrong** | Sequences, karma invisibility | Long-form, epistemic hygiene |
| **Ethereum Names** | Self-sovereign identity | Wallet as identity |
| **Mastodon** | Federation, no single owner | Decentralized community |
| **Nervos** | "Common knowledge" blockchain | On-chain reputation |
| **Urbit** | Persistent identity, deterministic IDs | Identity as property |
