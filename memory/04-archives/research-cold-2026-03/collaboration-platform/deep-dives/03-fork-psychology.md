# Fork Psychology - Deep Dive

## Why Forks Happen

### Triggers
1. **Disagreement on direction** - Can't reach consensus
2. **Governance capture** - Feel voiceless
3. **Personality conflict** - Can't work with current leaders
4. **Ideological split** - Fundamental value difference
5. **Exit signal** - Last resort before leaving

### Fork Types

| Type | Description | Example |
|------|-------------|---------|
| **Hard Fork** | Complete split, new community | Ethereum → Ethereum Classic |
| **Soft Fork** | Disagreement but stay connected | Bitcoin segwit |
| **Micro-fork** | Single proposal alternative | Branch sub-topic |

## The Fork Experience

### Emotional Journey
```
Denial → Frustration → Debate → Impasse → Fork → Relief/Anxiety
```

### Stages
1. **Early Warning** - Growing dissatisfaction
2. **Voicing Concerns** - Try to change from within
3. **Finding Allies** - Connect with like-minded
4. **Pre-fork Discussion** - Prepare the split
5. **The Fork** - Physical split of resources
6. **Post-fork** - Both sides evaluate outcome

## Mitigating Fork Problems

### Design Principles

1. **Low-cost Exit** - Can leave without burning bridges
2. **Voice Channels** - Ensure all can speak
3. **Graduated Consensus** - Don't require unanimity
4. **Regular Check-ins** - Catch issues early
5. **Fork as Feature** - Embrace it as natural

## Case Studies

### Successful Forks
- **Linux → Android** - Both thriving
- **Reddit (r BTC vs BCH)** - Both communities exist
- **Mastodon instances** - Federation works

### Failed Forks
- **Ethereum DAO** - Lost community trust
- **dy/dn** - Creator burnout

## Design Recommendations

1. **Make forks easy** - Don't penalize leaving
2. **Inherit history** - New branch gets parent history
3. **Bridge contributions** - Reference original
4. **Easy return** - Can rejoin if circumstances change
5. **Fork analytics** - Track fork reasons for learning
