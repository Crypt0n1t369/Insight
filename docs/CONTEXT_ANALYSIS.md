## BEST PRACTICES (Research Summary)

### 1. Context Window Management
| Practice | Description |
|----------|-------------|
| **Streaming Summaries** | Summarize at 50% capacity, replace old messages with summary |
| **Importance Filtering** | Keep system prompts, key decisions; drop verbose tool outputs |
| **Structural Truncation** | Trim tool results, keep only relevant portions |
| **Tiered Memory** | Hot (current session) / Warm (recent) / Cold (archived) |

### 2. Session Persistence
| Practice | Description |
|----------|-------------|
| **Memory Injection** | Pull relevant context from long-term store on startup |
| **Session State Files** | Persist state to disk, reload on restart |
| **Pre-prompt Templates** | Standardized session initialization |
| **Continuity Markers** | Reference previous sessions explicitly |

### 3. Continuity Optimization
| Practice | Description |
|----------|-------------|
| **Semantic Caching** | Cache common queries/responses |
| **Project Context Files** | Dedicated CONTEXT.md per project |
| **Decision Logs** | Track why decisions were made |
| **Progress Trackers** | Single source of truth for project state |

### 4. Automation Requirements
| Practice | Description |
|----------|-------------|
| **Cron-triggered Refresh** | Auto-update context before sessions |
| **Health-triggered Actions** | Context rebuild on threshold |
| **Startup Injection** | Load context before first response |
| **Budget-aware Scaling** | Switch models based on task complexity |

## GAPS IDENTIFIED

### Gap 1: No Auto-Injection on Startup
**Problem:** Even though `.memory_context` is generated, it's not read into each session's context window.

**Solution:** Wire `.memory_context` into the session initialization.

### Gap 2: No Auto-Summarization
**Problem:** No automatic compression when context fills.

**Solution:** Implement summarization at 50% threshold.

### Gap 3: No True Cross-Session Continuity
**Problem:** Must manually call `memory_recall` to find past context.

**Solution:** Auto-load relevant memories based on current session keywords.

### Gap 4: No Structured Decision Log
**Problem:** Decisions scattered across memory files.

**Solution:** Extract decisions to `DECISIONS.md` with timestamps.
