# Phase 1 MVP Runbook (Phase 0 → Phase 1)

## Objective
Establish Phase 1 MVP backbone: identity, operating model, memory/KB scaffold, model-tier policy, and a skeleton HEARTBEAT with guardrails. Ensure security, cost awareness, and auditable traces.

## Scope
- Identity and operating model (SOUL.md, IDENTIY.md)
- Memory backbone (MEMORY + memory/YYYY-MM-DD)
- Phase 1 MVP model-tier policy (AGENTS)
- Heartbeat skeleton
- Git repo scaffolding (main/develop/feature/*)
- 5 core tests (planning, ingestion, KB, urgent routing, security gate)

## Success criteria
- Phase 0 foundations exist and are functional
- Phase 1 MVP backbone implemented and testable
- Budget guardrails enforced in test harness
- Complete changelog and memory notes created

## Milestones
- M0: Phase 0 foundations committed
- M1: Phase 1 MVP backbone implemented
- M2: MVP test harness added and exercised
- M3: Phase 1 review ready

## Rollback plan
- If Phase 1 cannot be completed within time/budget, revert to Phase 0 baseline state; preserve a readme of changes and why it failed
